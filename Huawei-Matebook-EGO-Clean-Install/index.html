Huawei Matebook EGO Clean Install

Posted on 2025-03-09 Edited on 2025-03-10 In Tutorial
清洁重装Matebook E Go，同时也可以给其他Windows ARM的笔记本参考。

配置

  • Device: HUAWEI MateBook E Go 2023 (16GB+512GB)
  • CPU: Snapdragon (TM) 8cx Gen 3 @ 2.69 GHz
  • System: Windows 11 Pro 24H2 (Arm64)
  • Pen: HUAWEI M-Pencil 3

前置要求

  • 一台可用的Windows电脑
  • USB Flash Drive with 8GB at least
  • Windows 11 Arm64 ISO
  • 制作带驱动的镜像：DISM++
  • 制作启动盘：Rufus
  • 调整启动盘的分区大小：IM-Magic Partition Resizer Free

备份驱动

1.如果Matebook EGO已经无法使用，可以下载boot.wim（出自GitHub），然后Dism++挂载映像；否则无须这一步。

2.导出驱动

驱动管理 > 全选 > 导出驱动

3.删掉没用的驱动（洁癖可选）

在存放导出驱动的文件夹里搜索.sys和.dll文件，然后调整视图为详细信息，并在详细信息里勾选显示版权，就可以看到文件的签名。可以把除了高通和微软以外的驱动所在的文件夹删掉。

建议同时保存一份完整的驱动，以备不时之需。

但注意以下的不能删除：

  • 指纹：生物识别设备/gfspi.inf_arm64_8965dda759768c54
  • 触屏：未知/hidinjectorthp.inf_arm64_87733add234fc442
  • 触屏：未知/spbthptool.inf_arm64_f56d385517546dd1

完事后 DISM++ > 文件 > 卸载映像

导入驱动

1.解压或者挂载Windows 11 Arm64 ISO

可以看到sources文件夹里有两个boot.wim和install.wim两个wim文件。其中的boot.wim是用于启动安装镜像以及Windows系统修复用的，安装过程中会被解压到recovery分区；install.wim则是实际的系统安装包。

2.导入驱动并保存

挂载boot.wim > 导入驱动

等待导入完成后，文件 > 另存为映像。

保存完成后，文件 > 卸载映像。

对install.wim同理。我在boot.wim中导入的是完整版驱动；在install.wim中则删除了无用驱动。

备份其他驱动

如果Matebook EGO已经不可用，可以暂时先跳过这一步，继续重装系统。这样重装系统后会导致触屏和笔不可用，可以去官网下载软件管家。软件管家会帮你重装这部分驱动，然后再来做这一步的备份，最后再重新安装一遍系统来保持干净。

在Windows服务中可以看到一个HuaweiThpService，这个服务是触屏和笔的组件之一。

图里可以看到它的路径是”C:/Programs Files/HuaweiThpService”，把这个文件夹备份下来。

制作安装盘

如果是Linux或者有旧版Windows系统的，这一步应该可以完全用命令行解决。Win11限制了对GPT EFI分区的访问，我在使用diskpart命令时会出现错误，所以不得不依赖其他工具。

1.把Windows 11 Arm64 ISO的文件复制到一个新文件夹，除了setup.exe和sources文件夹不用复制；然后自己创建一个sources文件夹，并把ISO里的sources/boot.wim复制到里面。最终得到的目录结构如下：

然后DISM++ > 工具箱 > ISO生成器，把这个目录做成ISO。

2.用rufus把上一步的ISO写入U盘，注意选项：分区类型GPT，文件系统FAT32，写入方式ISO镜像。

这些选项可能因机器而异，我的Matebook EGO无法识别MBR分区或者NTFS文件系统的启动盘，这导致了我得把大小超过4GB的install.wim放到另一个分区，也就需要接下来的一步。

3.用IM-Magic Partition Resizer Free调整分区大小，如果有多个U盘的也没必要做这一步。

先把原有的rufus创建的分区调整为2048MB（我试了1024MB，此时我的电脑没法识别到启动盘），然后再创建一个大约6GB大小（或者剩余的全部也行）的NTFS分区用于存放sources文件夹的内容。由于我的U盘比较大，就把剩余的空间做了第三个分区。

4.把Windows 11 Arm64 ISO的sources文件夹的内容，除了boot.wim和install.wim以外放到6GB的NTFS分区中，然后把自己用DISM++制作的boot.wim和install.wim放到进去。（这里的boot.wim和上面自制ISO的boot.wim是同一个，但是不能不放，否则会导致安装失败。）

5.可以把备份的其他驱动比如ThpService放到6G的NTFS分区里，并不会影响安装。等安装完成后可以拿出来用。

到这里就完成了USB安装启动盘的制作了。

安装系统及之后

1.安装系统

BIOS设置：关机状态下同时长按电源和音量+键可以进入BIOS，而且在BIOS里竟然可以正常使用触屏和键盘。。只需要打开USB Boot Support即可，无需关闭Secure Boot。

进入启动盘：重启时连续点按键盘的Insert，选择USB启动。之后会看到一个控制台黑窗口，但是触屏和屏幕旋转都不能使用。使用命令行运行6GB的NTFS分区中的setup.exe即可进入安装程序。

（不需要）绕过安装需求检查：控制台窗口里输入regedit即可打开注册表，也可以用只用命令行。

跳过联网：Shift+F10可以打开控制台，然后输入oobe/bypassnro，等待重启即可。

2.安装设备

如果一切顺利，到这里系统成功，但除了触屏和笔不可用。

正常在设备管理器的THP设备可以看到以下3个：

安装完成后有可能会缺少THP VHF Device这一项，需要手动安装：

  • Action > Add legacy hardware
  • Install the hardware that I manually select from a list (Advanced)
  • THP devices

完事后应该就可以看到3个Thp设备。

3.添加触屏服务

把备份好的HuaweiThpService放到”C:/Programs Files/“路径下。

以管理员身份运行CMD，输入以下命令：

sc create HuaweiThpService binpath="C:/Programs Files/HuaweiThpService/HuaweiThpService.exe" start=auto

重启即可。

4.笔连接

在Microsoft Store下载安装HuaweiPenAPP即可。

但我实际使用中存在问题：尽管能够正常使用，该软件将我的3代笔识别为1代；并且在每次连接时会和键盘冲突，导致键盘巨卡，需要电源键关闭屏幕（睡眠）等键盘闪烁重置后才行。
